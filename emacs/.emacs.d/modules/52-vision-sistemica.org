* 10. Infraestructura de Métricas y Dashboards
Crea la estructura de directorios y archivos para el análisis de procesos.
El directorio base es =~/org/procesos/=.

Filosofía (Goldratt): "Decime cómo me medís y te diré cómo me comporto."
Sin métricas visibles, no hay mejora posible.

#+begin_src emacs-lisp
;; --- Directorio base de procesos ---
(defvar my/proc-dir (expand-file-name "procesos/" org-directory)
  "Directorio raíz de visión sistémica y métricas.")

(defvar my/proc-dashboard  (expand-file-name "dashboard.org"    my/proc-dir))
(defvar my/proc-vsm        (expand-file-name "vsm.org"          my/proc-dir))
(defvar my/proc-restricc   (expand-file-name "restricciones.org" my/proc-dir))
(defvar my/proc-muda       (expand-file-name "desperdicios.org"  my/proc-dir))
(defvar my/proc-comms      (expand-file-name "comunicaciones.org" my/proc-dir))
(defvar my/proc-diagramas  (expand-file-name "diagramas/"       my/proc-dir))

;; Crear directorios si no existen
(dolist (dir (list my/proc-dir my/proc-diagramas))
  (unless (file-directory-p dir)
    (make-directory dir t)
    (message "📁 Creado: %s" dir)))

;; --- Archivos semilla ---
(defun my/proc-ensure-file (filepath title &optional content)
  "Crea FILEPATH con TITLE y CONTENT opcional si no existe."
  (unless (file-exists-p filepath)
    (with-temp-file filepath
      (insert (format "#+TITLE: %s\n" title))
      (insert "#+FILETAGS: :procesos:\n")
      (insert "#+STARTUP: indent overview\n\n")
      (when content (insert content)))
    (message "📄 Creado: %s" filepath)))

(my/proc-ensure-file my/proc-dashboard "Dashboard de Métricas"
  "* KPIs del Servicio\n* Tendencias\n* Alertas\n")

(my/proc-ensure-file my/proc-vsm "Value Stream Maps"
  "* Mapas de Flujo de Valor\nCada sección documenta un proceso de punta a punta.\n")

(my/proc-ensure-file my/proc-restricc "Registro de Restricciones (TOC)"
  "* Restricciones Identificadas\n| Fecha | Proceso | Restricción | Estado | Acción |\n|-------+---------+-------------+--------+--------|\n|       |         |             |        |        |\n\n* Historial de Mejoras\n")

(my/proc-ensure-file my/proc-muda "Registro de Desperdicios (MUDA)"
  "* Los 7 Desperdicios — Registro de Observaciones
| Fecha | Tipo de Desperdicio | Dónde se observó | Impacto estimado | Acción propuesta |
|-------+---------------------+------------------+------------------+------------------+
|       |                     |                  |                  |                  |

** Referencia: Los 7 Desperdicios (Ohno / TPS)
1. *Sobreproducción*: Hacer más de lo que el cliente/usuario pidió
2. *Espera*: Tiempo muerto entre pasos del proceso
3. *Transporte*: Mover cosas (archivos, mails, equipos) innecesariamente
4. *Sobreprocesamiento*: Pasos que no agregan valor al resultado
5. *Inventario*: Acumular trabajo en progreso sin terminarlo
6. *Movimiento*: Desplazamientos innecesarios (del técnico, del usuario)
7. *Defectos*: Errores que requieren retrabajo
")

(my/proc-ensure-file my/proc-comms "Registro de Comunicaciones Técnicas"
  "* Comunicaciones Enviadas\n* Templates Utilizados\n")

* 20. Tablero Kanban Visual (org-kanban)
Un tablero Kanban hace visible el flujo de trabajo: qué está pendiente,
qué está en proceso, qué está terminado. Es la herramienta central de
los Tres Caminos de DevOps (Kim) y del Sistema Toyota (Ohno).

Regla WIP (Work In Progress): limitar el trabajo en curso para evitar
cuellos de botella. Si hay demasiado en "En Progreso", el sistema está
sobrecargado.

org-kanban genera un tablero a partir de los estados TODO del archivo.

#+begin_src emacs-lisp
(use-package org-kanban
  :defer t
  :config
  ;; Usar los estados de 50-gtd.org como columnas
  (setq org-kanban/layout '("..." . 30))) ; Ancho de columna

;; --- Funciones para insertar tableros Kanban ---
(defun my/proc-insertar-kanban ()
  "Inserta un bloque de tablero Kanban dinámico en el punto actual."
  (interactive)
  (insert "\n#+BEGIN: kanban :mirrored t :range (\"/\" . \"/\")\n")
  (insert "#+END:\n")
  (forward-line -1)
  (org-ctrl-c-ctrl-c)
  (message "📊 Tablero Kanban insertado. Actualizar con C-c C-c"))

;; --- Función para visualizar WIP (Work In Progress) ---
(defun my/proc-wip-status ()
  "Muestra el conteo de tareas por estado en los archivos de agenda."
  (interactive)
  (let ((counts '())
        (files org-agenda-files))
    (dolist (file files)
      (when (file-exists-p file)
        (with-current-buffer (find-file-noselect file)
          (org-map-entries
           (lambda ()
             (let ((state (org-get-todo-state)))
               (when state
                 (let ((pair (assoc state counts)))
                   (if pair
                       (setcdr pair (1+ (cdr pair)))
                     (push (cons state 1) counts))))))))))
    (let ((msg ""))
      (dolist (pair (sort counts (lambda (a b) (string< (car a) (car b)))))
        (setq msg (concat msg (format "  %s: %d" (car pair) (cdr pair)))))
      (message "📊 WIP:%s" msg))))
#+end_src

* 30. Value Stream Mapping con Mermaid y Graphviz
El Value Stream Mapping (VSM) identifica cada paso de un proceso,
midiendo tiempo de valor agregado vs. tiempo de espera.
Permite detectar cuellos de botella y desperdicios visualmente.

Usamos dos motores de diagramas:
- *Mermaid*: sintaxis simple, ideal para flujos y diagramas de secuencia
- *Graphviz (dot)*: más potente, ideal para grafos complejos y VSMs

Ambos se ejecutan dentro de bloques org-babel y generan imágenes inline.

Requisitos del sistema:
#+begin_src bash :tangle no
# Mermaid CLI (requiere npm, ya instalado desde 00-core)
sudo npm install -g @mermaid-js/mermaid-cli

# Graphviz
sudo pacman -S graphviz
#+end_src

* 40. Métricas e Indicadores (KPI) con Org Tables
Las tablas de Org Mode son spreadsheets embebidos: soportan fórmulas,
referencias, y se recalculan con =C-c C-c=.

KPIs centrales para un servicio de soporte (ITIL):
- *MTTR* (Mean Time To Resolve): promedio de tiempo de resolución
- *SLA Compliance*: % de tickets resueltos dentro del SLA
- *First Contact Resolution*: % resueltos en primer contacto
- *Backlog*: tickets abiertos acumulados
- *Satisfacción del usuario*: encuesta post-resolución

#+begin_src emacs-lisp
;; --- Insertar tabla de KPI con fórmulas ---

  (insert "|--------+-------------------+-----------+---------+-----------+-------+-------|\n")

  (insert "|      2 |                   |           |         |           |       |       |\n")
  (insert "|      3 |                   |           |         |           |       |       |\n")
  (insert "|      4 |                   |           |         |           |       |       |\n")

* 70. Comunicación Técnica Escrita (Templates)
El Módulo 3 incluye comunicación técnica: mails de escalado,
reportes de incidentes, informes semanales. Estas funciones generan
templates pre-estructurados que el alumno completa.

Principio (Kim, El Proyecto Fénix): "El flujo de información es tan
importante como el flujo de trabajo."

#+begin_src emacs-lisp
;; --- Template de mail de escalado ---
(defun my/proc-template-escalado ()
  "Genera un template de email de escalado técnico."
  (interactive)
  (let* ((ticket (read-string "Número de ticket: "))
         (asunto (read-string "Asunto breve: "))
         (nivel (completing-read "Escalar a: " '("N2" "N3" "Proveedor externo") nil t)))
    (switch-to-buffer (generate-new-buffer (format "*Escalado-%s*" ticket)))
    (org-mode)
    (insert (format "#+TITLE: Escalado %s — %s\n\n" ticket asunto))
    (insert (format "Para: Equipo %s\n" nivel))
    (insert (format "Asunto: [ESCALADO] Ticket #%s — %s\n\n" ticket asunto))
    (insert "---\n\n")
    (insert (format "Estimado equipo %s,\n\n" nivel))
    (insert (format "Escalo el ticket #%s por los siguientes motivos:\n\n" ticket))
    (insert "** Resumen del Problema\n")
    (insert "- Usuario afectado: \n")
    (insert "- Equipo: \n")
    (insert "- Inicio del incidente: \n\n")
    (insert "** Acciones Realizadas (N1)\n")
    (insert "1. \n")
    (insert "2. \n")
    (insert "3. \n\n")
    (insert "** Motivo del Escalado\n")
    (insert "- \n\n")
    (insert "** Información Adicional\n")
    (insert "- Logs adjuntos: \n")
    (insert "- Screenshots: \n\n")
    (insert "Quedo atento a cualquier consulta.\n\n")
    (insert "Saludos,\n")
    (insert (format "%s\nSoporte Técnico N1\n" (or user-full-name "Nombre")))
    (goto-char (point-min))
    (search-forward "Usuario afectado: ")
    (message "📧 Template de escalado generado. Completar y enviar.")))

;; --- Template de reporte semanal de servicio ---
(defun my/proc-template-reporte-semanal ()
  "Genera un template de reporte semanal del área de soporte."

* 80. Clock Reports: Tiempo y Flujo
Org Mode tiene un sistema de clock-in/clock-out integrado que permite
medir cuánto tiempo se dedica a cada tarea. Esto genera datos reales
para calcular MTTR, detectar tareas que consumen demasiado tiempo,
y alimentar los KPIs del dashboard.

Atajos: =C-c C-x C-i= (clock-in), =C-c C-x C-o= (clock-out)

#+begin_src emacs-lisp
;; --- Configuración de clocking ---
(setq org-clock-persist 'history)       ; Recordar el clock entre sesiones
(setq org-clock-in-resume t)            ; Reanudar clock al re-clock-in
(setq org-clock-out-remove-zero-time-clocks t) ; Limpiar clocks vacíos
(setq org-clock-report-include-clocking-task t) ; Incluir tarea actual en reportes

;; Ingerir historial de clocks una vez que org-clock esté cargado
(with-eval-after-load 'org-clock
  (org-clock-persistence-ingest))

;; --- Función para generar clock report del período ---
(defun my/proc-insertar-clock-report ()
  "Inserta un reporte de tiempo dedicado por tarea."
  (interactive)
  (let ((scope (completing-read "Alcance: "
                '("subtree" "file" "agenda") nil t "agenda"))
        (periodo (completing-read "Período: "
                  '("today" "thisweek" "lastweek" "thismonth") nil t "thisweek")))
    (insert (format "\n#+BEGIN: clocktable :scope %s :block %s :maxlevel 3 :emphasize t :link t\n"
                    scope periodo))
    (insert "#+END:\n")
    (forward-line -1)
    (org-ctrl-c-ctrl-c)
    (message "⏱️ Clock report insertado.")))

;; --- Función para calcular MTTR desde tickets cerrados ---
(defun my/proc-calcular-mttr ()
  "Calcula el MTTR promedio de tickets DONE en soporte.org."
  (interactive)
  (let ((total-time 0)
        (count 0))
    (when (file-exists-p my/gtd-soporte)
      (with-current-buffer (find-file-noselect my/gtd-soporte)
        (org-map-entries
         (lambda ()
           (when (string= (org-get-todo-state) "DONE")
             (let ((clock-minutes (org-clock-sum-current-item)))
               (when (> clock-minutes 0)
                 (setq total-time (+ total-time clock-minutes))
                 (setq count (1+ count)))))))))
    (if (> count 0)
        (let* ((avg-min (/ (float total-time) count))
               (avg-hrs (/ avg-min 60.0)))
          (message "📊 MTTR: %.1f horas (%.0f min) — basado en %d tickets cerrados con clock"
                   avg-hrs avg-min count))
      (message "No se encontraron tickets DONE con tiempo registrado."))))
#+end_src

(defun my/proc-ai-analizar-restricciones ()
  "Analiza el texto del subtree actual buscando restricciones (TOC)."
  (interactive)
